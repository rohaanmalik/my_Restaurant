What To Push Into Your Group Repository for Phase 1

  • Your code, organized into packages, with complete Javadoc and any internal comments that you think will be useful.

  • A file called README.txt that contains instructions about how to run your program. The TA will use this to run and mark your code.

  • A file called events.txt that simulates all input (examples: server enters order, cook confirms receipt of order, cook confirms that order has been prepared, etc.)

  • Any configuration files that are needed to start your program. They will contain any information that your program needs in order to run. These files are optional.

  • An up-to-date UML diagram that accurately reflects the code you push. The diagram will be in a file called design.pdf. You may want to set an earlier deadline for your code so that you have at least a day to create the UML diagram.

You will push all of these items to the phase1 repository for your group.
